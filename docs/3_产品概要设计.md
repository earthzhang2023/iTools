# iTools - 产品概要设计文档

## 1. 文档概述

### 1.1 文档目的
本文档描述 iTools 纯前端在线工具网站的整体架构设计、技术选型、模块划分和核心设计决策，为后续详细设计和开发实施提供指导。

### 1.2 适用范围
- 系统架构设计
- 技术栈选型
- 模块划分
- 核心设计决策
- 部署方案

| 术语 | 定义 |
|------|------|
| MVP | Minimum Viable Product，最小可行产品 |
| SSR | Server-Side Rendering，服务端渲染 |

### 2.2 架构特点
- **纯前端架构**：所有数据处理在浏览器本地完成，无需后端服务

- **模块化结构**：按功能模块划分，清晰的文件组织
- **响应式布局**：支持 PC、平板、手机多端访问
- **主题系统**：支持亮色/深色模式切换

### 3.3 技术选型理由

#### 3.3.1 为什么选择 Vue 3？
- **学习曲线平缓**：相比 React，Vue 更容易上手
- **生态成熟**：丰富的组件库和工具链
- **性能优异**：Composition API 提供更好的性能
- **TypeScript 支持**：原生支持 TypeScript
- **参考案例**：IT-Tools、程序员工具箱等成功案例

#### 3.3.2 为什么选择 Naive UI？
- **组件丰富**：提供 80+ 个高质量组件
- **暗黑模式**：原生支持主题切换
- **TypeScript 支持**：完整的类型定义
- **性能良好**：按需引入，体积可控
- **设计现代**：符合现代审美

#### 3.3.3 为什么选择 CodeMirror 6？
- **轻量**：模块化设计，体积可控
- **性能好**：支持大文件编辑
- **暗黑模式**：主题支持良好
- **扩展性强**：丰富的插件生态

## 5. 核心设计决策

### 5.1 纯前端架构
**决策**：所有数据处理在浏览器本地完成，不依赖后端服务。

**理由**：
- 保护用户隐私，数据不上传服务器
- 降低运维成本，无需后端服务器
- 响应速度快，无网络延迟
- 易于部署，静态托管即可

**影响**：
- 需要合理使用浏览器 API（FileReader、Canvas 等）
- 大文件处理需要考虑性能（使用 Web Worker）
- 需要处理浏览器兼容性问题

### 5.2 即时响应设计
**决策**：用户输入时实时更新输出，无需点击提交按钮。

**理由**：
- 提升用户体验，减少操作步骤
- 符合现代工具类网站趋势
- 技术上可行，性能影响小

**实现**：
- 使用防抖处理（300ms）避免频繁计算
- 耗时操作显示 Loading 状态
- 大文件处理使用 Web Worker

### 5.3 组件化设计
**决策**：每个工具独立组件，便于扩展和维护。

**理由**：
- 代码组织清晰
- 便于团队协作
- 易于扩展新工具
- 支持按需加载

**实现**：
- 统一的工具组件接口
- 工具注册机制
- 动态路由加载

### 5.4 主题系统设计
**决策**：使用 CSS 变量 + Naive UI 主题配置实现主题切换。

**理由**：
- Naive UI 原生支持主题
- CSS 变量便于自定义
- 支持运行时切换

**实现**：
- 使用 Pinia 管理主题状态
- 监听系统主题变化
- 主题切换动画过渡

### 5.5 响应式布局
**决策**：使用 Flexbox + Grid 实现响应式布局。

**理由**：
- 现代布局方案
- 浏览器支持良好
- 代码简洁

**实现**：
- 三个响应式断点（1280px、768px）
- 移动端优先策略
- 使用 CSS 媒体查询

## 6. 数据流设计

### 6.1 状态管理
```
用户输入 → 工具组件 → 工具函数 → 更新状态 → 渲染输出
                ↓
            Pinia Store（全局状态）
                ↓
          localStorage（持久化）
```

### 6.2 工具调用流程
```
1. 用户选择工具
2. 加载工具组件
3. 用户输入数据
4. 工具函数处理（防抖）
5. 更新输出结果
6. 用户复制结果
```

### 6.3 主题切换流程
```
1. 用户点击主题切换按钮
2. 更新 Pinia 主题状态
3. 更新 CSS 变量
4. 更新 Naive UI 主题配置
5. 保存到 localStorage
6. 触发主题切换动画
```

## 7. 性能优化策略

### 7.1 代码层面
- **按需引入**：组件和库按需引入
- **懒加载**：大型组件（如编辑器）懒加载
- **Tree Shaking**：移除未使用代码
- **代码分割**：按路由和模块分割代码

### 7.2 运行时优化
- **防抖节流**：输入处理使用防抖
- **缓存机制**：工具结果缓存
- **Web Worker**：大文件处理使用后台线程
- **虚拟滚动**：长列表使用虚拟滚动

### 7.3 资源优化
- **图片优化**：使用 WebP 格式
- **字体优化**：使用系统字体
- **CDN 加速**：第三方库使用 CDN
- **Gzip 压缩**：启用 Gzip 压缩

### 7.4 性能指标
- 页面加载时间 < 1.5 秒（LCP）
- 工具响应时间 < 100ms
- 页面总包体积 < 500KB（Gzip 后）
- 首次内容绘制 < 1.0 秒

## 8. 安全设计

### 8.1 数据安全
- 所有数据在浏览器本地处理
- 不发送任何用户数据到服务器
- 不使用第三方分析工具
- 不使用 Cookie 跟踪

### 8.2 代码安全
- 使用 HTTPS 协议
- 启用 CSP（内容安全策略）
- 防止 XSS 攻击
- 防止点击劫持

### 8.3 依赖安全
- 定期更新依赖
- 使用可靠的 npm 包
- 检查依赖安全漏洞
- 锁定依赖版本
