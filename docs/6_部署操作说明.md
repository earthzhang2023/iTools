## 准备工作

### 1. 注册账号

- **GitHub 账号**：访问 [https://github.com](https://github.com) 注册账号
- **Vercel 账号**：访问 [https://vercel.com](https://vercel.com) 注册账号（推荐使用 GitHub 账号直接登录）

### 2. 安装 Git

如果尚未安装 Git，请访问 [https://git-scm.com](https://git-scm.com) 下载并安装。

安装完成后，在终端中验证：

```bash
git --version
```

---

## 步骤一：创建 GitHub 仓库

### 1.1 创建新仓库

1. 登录 GitHub
2. 点击右上角的 **+** 按钮，选择 **New repository**
3. 填写仓库信息：
   - **Repository name**: `itools`（或其他你喜欢的名称）
   - **Description**: `iTools - 纯前端在线工具网站`
   - **Visibility**: 选择 **Public**（公开）或 **Private**（私有）
   - **不要** 勾选 "Initialize this repository with a README"
4. 点击 **Create repository**

### 1.2 记录仓库地址

创建成功后，复制仓库地址，格式如下：

```
https://github.com/你的用户名/itools.git
```

---

## 步骤二：推送代码到 GitHub

### 2.1 初始化本地仓库

在项目根目录（`d:\TraeProjects\iTools`）打开终端，执行：

```bash
git init
```

### 2.2 添加所有文件

```bash
git add .
```

### 2.3 提交代码

```bash
git commit -m "Initial commit: iTools v1.0"
```

### 2.4 关联远程仓库

将 `<你的仓库地址>` 替换为步骤 1.2 中复制的地址：

```bash
git remote add origin https://github.com/你的用户名/itools.git
```

### 2.5 推送到 GitHub

```bash
git branch -M main
git push -u origin main
```

> **注意**：如果使用 HTTPS 推送时遇到认证问题，可能需要使用 GitHub Token 或配置 SSH 密钥。

## 步骤三：在 Vercel 导入项目

### 3.1 登录 Vercel

1. 访问 [https://vercel.com](https://vercel.com)
2. 点击 **Sign Up** 或 **Log In**
3. 选择 **Continue with GitHub** 登录

### 3.2 导入项目

1. 登录后，点击 **Add New...** > **Project**
2. 在 **Import Git Repository** 页面，找到你的 `itools` 仓库
3. 点击 **Import**

---

## 步骤四：配置部署选项

### 4.1 项目配置

在 **Configure Project** 页面，配置以下选项：

- **Framework Preset**: 选择 **Vite**
- **Build Command**: `npm run build`
- **Output Directory**: `dist`
- **Install Command**: `npm install`

### 4.2 环境变量（如需要）

如果项目需要环境变量，点击 **Environment Variables** 添加。

> iTools 为纯前端项目，通常不需要配置环境变量。

### 4.3 点击部署

确认配置无误后，点击 **Deploy** 按钮。

---

## 步骤五：完成部署

### 5.1 等待构建

Vercel 会自动开始构建，过程如下：

1. 安装依赖
2. 执行构建命令
3. 部署到全球 CDN

构建过程通常需要 1-3 分钟。

### 5.2 部署成功

构建成功后，会显示：

- ✅ **Congratulations!**
- 项目访问地址：`https://itools-xxx.vercel.app`

点击 **Visit** 按钮即可访问你的网站。

### 5.3 查看部署详情

在 Vercel 仪表板中可以查看：

- 部署历史
- 构建日志
- 访问分析
- 域名设置

## 自定义域名（可选）

### 5.1 添加自定义域名

1. 在 Vercel 项目页面，点击 **Settings** > **Domains**
2. 点击 **Add** 按钮
3. 输入你的域名，例如：`itools.example.com`
4. 点击 **Add**

### 5.2 配置 DNS

根据 Vercel 提示，在你的域名服务商处配置 DNS：

- **类型**: `CNAME`
- **名称**: `itools`（或 `@`）
- **值**: `cname.vercel-dns.com`

### 5.3 等待生效

DNS 生效通常需要几分钟到几小时不等。

## 自动部署

### 开启 CI/CD

Vercel 已自动开启 CI/CD，当你推送代码到 GitHub 仓库时：

```bash
git add .
git commit -m "修复 bug"
git push
```

Vercel 会自动检测更改并重新部署，无需手动操作。

### 部署预览

对于 Pull Request，Vercel 会生成预览部署，方便代码审查。

## 常见问题

### Q1: 构建失败，提示 "Build Error"

**解决方案**：

1. 检查 `package.json` 中的 `build` 脚本是否正确
2. 查看构建日志，定位具体错误
3. 确保 `vite.config.js` 配置正确

### Q2: 部署后页面空白或 404

**解决方案**：

1. 检查路由配置是否使用 `createWebHistory()`
2. 在 Vercel 中添加 `vercel.json` 配置：

```json
{
  "rewrites": [
    { "source": "/(.*)", "destination": "/index.html" }
  ]
}
```

### Q3: GitHub 推送失败

**解决方案**：

- 使用 GitHub Token 代替密码
- 或配置 SSH 密钥：

```bash
ssh-keygen -t ed25519 -C "your_email@example.com"
# 将公钥添加到 GitHub Settings > SSH and GPG keys
```

### Q4: 如何更新项目？

**解决方案**：

在本地修改代码后：

```bash
git add .
git commit -m "更新说明"
git push
```

Vercel 会自动重新部署（约 1-3 分钟）。

## 项目维护

### 查看访问统计

1. 登录 Vercel
2. 进入项目页面
3. 点击 **Analytics** 查看访问数据

1. 进入 Vercel 项目
2. 点击 **Deployments**
3. 找到之前的稳定版本
4. 点击 **···** > **Promote to Production**
