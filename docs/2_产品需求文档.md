# 产品需求描述文档（PRD）- 纯前端在线工具网站

## 1. 项目背景
在开发者和日常办公人群中，存在大量对文本、数据、图片进行快速转换和处理的需求。传统工具类网站往往需要后端服务，存在数据隐私风险且维护成本高。纯前端工具网站利用浏览器本地 API 完成所有数据处理，无需上传服务器，具有**零隐私泄露**、**响应迅速**、**低成本维护**等核心优势。市面上已有 IT-Tools、JustHTMLs 等成功案例，本项目旨在构建一个轻量、现代、易用的在线工具集合。

## 2. 目标用户
- **开发者**：需要 JSON 格式化、Base64 编解码、时间戳转换、正则测试等日常开发辅助工具。
- **内容创作者 / 运营人员**：需要文本处理（大小写转换、字数统计）、Markdown 编辑器、二维码生成等功能。
- **普通办公人员**：需要颜色转换、单位换算、图片转 Base64 等简单实用工具。

## 3. 产品目标
- 提供一个**完全纯前端**、**无数据上传**的在线工具网站。
- 产品名称：**iTools**
- 作者：Dr.Zhang
- 网站Logo：产品名称+作者姓名
- 首期覆盖 **15~20 个高频工具**，后续可扩展。
- 具备**响应式设计**，支持 PC、平板、手机访问。
- 提供**深色/亮色主题**切换，提升用户体验。
- 所有工具**即输即得**，无需点击提交按钮（实时响应）。
- UI风格**简洁但炫酷**，符合现代审美。
- 部署采用Vercel方式部署，无需后端服务。

## 4. 功能需求（MVP 版本）

### 4.1 编码与解码模块
| 工具 | 描述 | 关键实现 |
|------|------|----------|
| Base64 编码/解码 | 文本与 Base64 双向转换 | `btoa()` / `atob()` |
| URL 编码/解码 | 对 URL 参数进行百分号编码 | `encodeURIComponent()` / `decodeURIComponent()` |
| Unicode 转换 | 中文 ↔ Unicode 转义序列 | `charCodeAt()` / `String.fromCharCode()` |
| HTML 实体编码/解码 | 转义 HTML 标签及特殊字符 | 正则替换 / DOM API |

### 4.2 JSON 处理模块
| 工具 | 描述 | 关键实现 |
|------|------|----------|
| JSON 格式化/压缩 | 美化或压缩 JSON 字符串 | `JSON.stringify()` + `JSON.parse()` |
| JSON 验证器 | 检测 JSON 语法错误并定位 | `try-catch` |
| JSON ↔ CSV 互转 | 支持简单二维 JSON 转 CSV | 递归遍历 + 字符串拼接 |
| JSON 转 TypeScript 接口 | 自动生成 TS 类型定义 | 遍历 JSON 结构生成 interface 字符串 |

### 4.3 时间与数字转换模块
| 工具 | 描述 | 关键实现 |
|------|------|----------|
| 时间戳转换 | Unix 时间戳（秒/毫秒）与日期时间互转 | `new Date()` API |
| 进制转换 | 二进制、八进制、十进制、十六进制互转 | `parseInt()` + `toString(radix)` |
| 颜色格式转换 | HEX ↔ RGB ↔ HSL 互转 | 正则 + 进制换算 |
| 文件大小换算 | B/KB/MB/GB 自动换算 | 数学计算 |

### 4.4 文本处理模块
| 工具 | 描述 | 关键实现 |
|------|------|----------|
| 大小写转换 | 全大写、全小写、首字母大写、句子大小写 | 字符串方法 + 正则 |
| 命名风格转换 | 驼峰、下划线、短横线、常量互转 | 正则 + 字符串处理 |
| 字符/单词统计 | 实时统计长度、单词数、行数、中文字符数 | 正则匹配 + 计数 |
| 文本去重/排序 | 去除重复行，按字母或长度排序 | 数组 `Set` / `sort()` |
| 正则表达式测试 | 输入正则和测试文本，显示匹配结果 | `RegExp` 对象 + `matchAll()` |

### 4.5 图片与媒体模块
| 工具 | 描述 | 关键实现 |
|------|------|----------|
| 图片转 Base64 | 上传图片，生成 DataURL 并预览 | `FileReader` |
| Base64 转图片 | 粘贴 Base64 字符串，显示图片并提供下载 | 设置 `<img>` 的 `src` |
| 二维码生成器 | 输入文本/URL，生成二维码图片 | QRCode.js 库 |
| 简易图片压缩 | 调整图片质量或尺寸（可选） | Canvas API |

### 4.6 加密与哈希模块
| 工具 | 描述 | 关键实现 |
|------|------|----------|
| MD5 哈希 | 计算文本的 MD5 值 | crypto-js 或 Web Crypto API |
| SHA-256 哈希 | 计算 SHA-256 哈希值 | Web Crypto API `subtle.digest` |
| JWT 解析器 | 解析 JWT 的三个部分（Header、Payload、Signature） | `atob()` 解码 |

## 5. 非功能需求

| 需求类型 | 描述 |
|----------|------|
| **性能** | 单个工具操作响应时间 < 100ms；大文件 JSON（10MB）格式化不卡死 UI（使用 Web Worker 可选） |
| **兼容性** | 支持 Chrome、Firefox、Safari、Edge 最近两个大版本；移动端主流浏览器 |
| **隐私性** | 所有处理在浏览器本地完成，**不发送任何用户数据到服务器**，无需网络请求 |
| **可用性** | 每个工具提供简短说明或示例；支持一键复制结果；输入区域支持清空 |
| **可访问性** | 遵循 WCAG 2.1 标准的基本要求，支持键盘操作 |
| **国际化** | MVP 版本支持中文/英文双语（自动检测或手动切换） |

## 6. 技术栈建议

| 层级 | 推荐方案 | 备选方案 |
|------|----------|----------|
| 前端框架 | Vue 3 + TypeScript + Vite | React + Next.js (SSG) |
| UI 组件库 | Naive UI（与 IT-Tools 相同，组件丰富，支持暗黑模式） | Element Plus / Ant Design |
| 代码编辑器（用于 JSON/文本工具） | CodeMirror 6（轻量，性能好） | Monaco Editor（功能强但体积大） |
| 状态管理 | Pinia（Vue）或 Context（React） | 无需全局状态，组件内管理亦可 |
| 样式方案 | CSS Modules + CSS 变量（主题切换） | Tailwind CSS |
| 部署 | Vercel / Netlify / GitHub Pages | 任意静态托管服务 |
| 二维码生成 | `qrcode` 库 | `QRCode.js` |
| 加密哈希 | Web Crypto API（原生，无需额外库） | `crypto-js` |

## 7. UI/UX 设计要求

- **布局**：左侧为工具导航菜单（按模块分组），右侧为当前工具的操作面板。移动端改为顶部汉堡菜单 + 全屏卡片。
- **主题**：支持亮色/深色模式切换，默认跟随系统。
- **网站Logo**：设计简洁，鲜明，产品名称+作者姓名，居中在右上角。
- **卡片风格**：圆角 16px，柔和阴影，白色背景（亮色模式），深色背景（暗黑模式），具有动画过渡效果。
- **交互反馈**：
  - 输入框内容变化时实时更新输出（防抖处理，避免频繁计算）。
  - 复制成功时显示 Toast 提示。
  - 对于耗时操作（如大 JSON 格式化），显示 Loading 状态。
- **响应式断点**：
  - ≥1280px：两栏布局（导航 + 内容）。
  - 768px~1279px：导航可折叠，内容区自适应。
  - <768px：导航抽屉式，内容区全宽。

## 8. 优先级规划

### Phase 1（核心功能，第 1~2 周）
- 项目脚手架搭建，主题切换实现
- JSON 格式化/验证器
- Base64 编码解码
- 时间戳转换
- URL 编码解码
- 文本大小写转换

### Phase 2（常用扩展，第 3~4 周）
- 二维码生成器
- MD5 / SHA-256 哈希
- 颜色转换（HEX/RGB/HSL）
- 正则表达式测试
- 进制转换
- 图片转 Base64

### Phase 3（进阶功能，第 5~6 周）
- JSON ↔ CSV 互转
- JSON 转 TypeScript 接口
- 文本去重/排序
- 命名风格转换
- 简易图片压缩（可选）

### Phase 4（优化与发布，第 7 周）
- 性能优化（大文件 Web Worker 处理）
- 多语言支持（中英文）
- 单元测试（核心工具函数）
- vercel部署文档与使用说明文档编写

## 9. 成功指标（KPI）

- 单个工具页面加载时间 < 1.5 秒（LCP）
- 工具交互响应时间（从输入到输出更新）< 100ms
- 页面总包体积（Gzip 后）< 500KB（不含编辑器库，编辑器可懒加载）
- 上线后一个月内累计独立访客 > 1000（通过自然搜索）
- 用户留存：次日回访率 > 15%
